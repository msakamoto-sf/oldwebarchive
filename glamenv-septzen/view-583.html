"Hello, World!"レベルの単純なプログラムを題材に、基本的なコンパイルオプションのまとめ。
対象：Visual C++ 2008 Express Edition 

> cl
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86
Copyright (C) Microsoft Corporation.  All rights reserved.
> link
Microsoft (R) Incremental Linker Version 9.00.30729.01
Copyright (C) Microsoft Corporation.  All rights reserved.

ここではコンソール/GUI/DLLに関わらず共通して使われそうなオプションをまとめてみた。

  • コマンドラインでコンパイルする基本(clコマンド)
  • 最適化無し(/Od), 著作権情報表示無し(/nologo), 全警告表示(/Wall, /Wn)
  • 複数モジュールの分割コンパイル(/c, LINKコマンド)
  • "C" or "C++"の指定(/TC, /TP)
  • コメント形式とMicrosoftのC/C++言語拡張機能の有効・無効切り替え(/Za, /Ze)
  • インクルードパスの調整(/I, /X)
  • プリプロセス結果を取得(コンパイルは行わない)(/P, /E, /EP, /C)
    • "/E"で標準出力へ
    • "/EP"で"#line"無しで標準出力へ＋コメント保持
    • "/P"で"#line"付きでファイルへ保存
    • "/P" + "/EP"で"#line"無しでファイル保存
  • アセンブラコードを出力(/FA)
    • /FA : シンプルな ".asm"
    • /FAc : マシン語付 ".cod"
    • /FAs : 元のソースコード付 ".asm"
    • /FAcs or /FAsc : マシン語＋元のソース付 ".cod"
  • その他使うかも知れないオプション

コマンドラインでコンパイルする基本(clコマンド)

シンプルな基本：

HelloWorld.c: 

#include <stdio.h>
 
int main()
{
    printf("Hello, World!\n");
    return 0;
}

コンパイル: 

> cl HelloWorld.c
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86
Copyright (C) Microsoft Corporation.  All rights reserved.

HelloWorld.c
Microsoft (R) Incremental Linker Version 9.00.30729.01
Copyright (C) Microsoft Corporation.  All rights reserved.

/out:HelloWorld.exe
HelloWorld.obj

実行: 

> HelloWorld.exe
Hello, World!

最適化無し(/Od), 著作権情報表示無し(/nologo), 全警告表示(/Wall, /Wn)

/Od
最適化無し
/nologo
著作権情報表示無し
/Wall, /Wn(nは数字)
"/Wall"で全警告を表示。あるいは"/W1" - "/W4"。IDEでWin32コンソールアプリのプロジェクトを作成すると、デフォルトは"/W3"になっている。

HelloWorld.c: 

#include <stdio.h>
 
int main(int argc, char *argv[])
{
    printf("Hello, World!\n");
    return 0;
}

コンパイル："/Wall" 

> cl /Od /nologo /Wall HelloWorld.c

HelloWorld.c(3) : warning C4100: 'argc' : 引数は関数の本体部で 1 度も参照されません。

コンパイル："/W3" 

> cl /Od /nologo /W3 HelloWorld.c
HelloWorld.c

複数モジュールの分割コンパイル(/c, LINKコマンド)

/c
".obj"ファイルの作成までで、リンクは行わない

main.c: 

#include <stdio.h>

extern int foo(int a, int b, int c, int d);
extern int bar(int a, int b, int c, int d);

int main(int argc, char *argv[])
{
	int r;
	r = foo(4, 3, 2, 1);
	printf("foo() = %d\n", r);
	r = bar(4, 3, 2, 1);
	printf("bar() = %d\n", r);
	return 0;
}

foo.c: 

int foo(int a, int b, int c, int d)
{
	int r;
	r = a + b + c + d;
	return r;
}

bar.c: 

int bar(int a, int b, int c, int d)
{
	int r;
	r = a - b - c - d;
	return r;
}

分割コンパイル：

> cl /c bar.c
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86
Copyright (C) Microsoft Corporation.  All rights reserved.

> cl /c main.c
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86

"C" or "C++"の指定(/TC, /TP)

デフォルトでは拡張子から自動判別する。次のオプションを指定することで、明示的に入力ソースファイルの言語が"C"か"C++"かを指定出来る。

/TC
コマンドラインで指定されたコンパイル対象のソースファイルを全てC言語として扱う
/TP
コマンドラインで指定されたコンパイル対象のソースファイルを全てC++言語dとして扱う

"/TC"のテスト版：HelloWorld_TCtest.foo: 

#include <stdio.h>

/* C style multi line comment */
// C++ style single line comment

int main(int argc, char *argv[]) {
	printf("Hello, World!\n");
	return 0;
}

コンパイル＆実行： 

> cl /TC HelloWorld_TCtest.foo

Copyright (C) Microsoft Corporation.  All rights reserved.

Copyright (C) Microsoft Corporation.  All rights reserved.

/out:HelloWorld_TCtest.exe
HelloWorld_TCtest.obj

Hello, World!

コメント形式とMicrosoftのC/C++言語拡張機能の有効・無効切り替え(/Za, /Ze)

VC++2008のC/C++コンパイラでは、デフォルトではMicrosoftによるC/C++言語拡張機能が有効化されている。
これにより、C言語としてコンパイルする場合でもC++形式の一行コメント、"//"が使えるようになっている。

/Za
MicrosoftによるC/C++言語拡張機能を無効にする。Cソースの場合、"//"は警告になる。
/Ze
MicrosoftによるC/C++言語拡張機能を有効にする。(デフォルト)

一行コメントの場合、エスケープシーケンスが行末にあると次の行もコメントとして認識するようになっている。ただし、実際に使ってみると、以下の警告が表示される。 

warning C4010: 単一行コメント (//) に、行連結文字があります。

また、"/Ze"の明示的な指定は現在推奨されていないらしい。下記に示すように今後のバージョンからは削除される旨の警告が表示される。

コメントスタイル検証用：HelloWorld_TCtest.foo: 

#include <stdio.h>

/* C style multi line comment */
// C++ style single line comment

int main(int argc, char *argv[]) {
	printf("Hello, World!\n");
	return 0;
}

"/Ze"でコンパイル:

                                     今後のバージョンからは削除されます。

Microsoft (R) Incremental Linker Version 9.00.30729.01

インクルードパスの調整(/I, /X)

/I
インクルードディレクトリを追加する。複数指定したい場合は、ディレクトリの分だけ "/I ディレクトリ名"を指定する。
/X
標準のインクルードディレクトリを無視する。

実験用のファイル構成： 

pptest.c
inc/
    pptest.h

pptest.c: 

#include "pptest.h"

/**
 * comment1
 */
int foo(int a, int b) {
	return a + b;
}

/**
 * comment2
 */
int bar(int a, int b) {
	return a * b;
}

pptest.h: 

#ifdef _DEBUG
char message[] = "Debug Message";
#else
char message[] = "Release Message";
#endif /* _DEBUG */

実験１："/I"を指定しない→コンパイルエラー(fatal error C1083) 

> cl /c pptest.c
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86
Copyright (C) Microsoft Corporation.  All rights reserved.

pptest.c(1) : fatal error C1083: include ファイルを開けません。'pptest.h': No such file or directory

実験２："/I"を指定→コンパイル成功 

> cl /c /I .\inc pptest.c
Microsoft(R) 32-bit C/C++ Optimizing Compiler Version 15.00.30729.01 for 80x86
Copyright (C) Microsoft Corporation.  All rights reserved.

pptest.c

> dir pptest.obj
(...)
2010/02/11  11:46               625 pptest.obj
               1 個のファイル                 625 バイト
(...)

実験３：pptest.cでstdio.hをインクルードさせた上で、"/X"で標準インクルードディレクトリを無視させる→コンパイルエラー(fatal error C1083)

プリプロセス結果を取得(コンパイルは行わない)(/P, /E, /EP, /C)

/P
"#line"付でプリプロセス出力がファイルに保存される。ファイル名はソースファイル名のbasename + ".i"。
/E
"#line"付でプリプロセス出力が標準出力に出力される。
/EP
"#line"無しでプリプロセス出力を標準出力に出力、あるいは"/P"との組み合わせでファイルに保存する。
/C
プリプロセス時にコメントを保持する。

「インクルードパスの調整」で使ったpptest.c, pptest.hを使って実験してみる。
※見やすくする為、プリプロセス結果中の連続する空行は一行に縮めています。

アセンブラコードを出力(/FA)

※いずれのオプションでもコンパイル以降の処理も続行される。

/FA
アセンブラコードのみの".asm"ファイルを生成
/FAc
対応するマシン語が埋め込まれたアセンブラコードを生成(拡張子：".cod")
/FAs
対応するソースコードが埋め込まれたアセンブラコードを生成(拡張子：".asm")
/FAcs or /FAsc
対応するマシン語＋対応するソースコードが埋め込まれたアセンブラコードを生成(拡張子：".cod")

実験ファイル：fatest.c: 

int foo(int a, int b, int c)
{
	int r;
	r = a + b + c;
	r *= 3;
	return r;
}

_a$ = 8							; size = 4
_b$ = 12						; size = 4
_c$ = 16						; size = 4
_foo	PROC

; 2    : {

  00000	55		 push	 ebp
  00001	8b ec		 mov	 ebp, esp
  00003	51		 push	 ecx

; 3    : 	int r;
; 4    : 	r = a + b + c;

  00004	8b 45 08	 mov	 eax, DWORD PTR _a$[ebp]
  00007	03 45 0c	 add	 eax, DWORD PTR _b$[ebp]
  0000a	03 45 10	 add	 eax, DWORD PTR _c$[ebp]
  0000d	89 45 fc	 mov	 DWORD PTR _r$[ebp], eax

; 5    : 	r *= 3;

  00010	8b 4d fc	 mov	 ecx, DWORD PTR _r$[ebp]
  00013	6b c9 03	 imul	 ecx, 3
  00016	89 4d fc	 mov	 DWORD PTR _r$[ebp], ecx

; 6    : 	return r;

  00019	8b 45 fc	 mov	 eax, DWORD PTR _r$[ebp]

; 7    : }

  0001c	8b e5		 mov	 esp, ebp
  0001e	5d		 pop	 ebp
  0001f	c3		 ret	 0
_foo	ENDP
_TEXT	ENDS
END

その他使うかも知れないオプション

/Fd
"/ZI"などデバッグ情報オプション指定時に生成されるプログラムデータベース(PDB)ファイル名を "/Fdpathname" で指定する。
/Fe
"/Fepathname"でコンパイラにより生成されるexe/dllのファイル名とディレクトリを指定する。"/c"オプション付きの場合は無効になる。
/Fd
"/Fdpathname"で".obj"ファイル名を指定する。
/Oy
フレームポインタの省略(x86専用)
/Z7, /Zi, /ZI
デバッグ情報の形式を指定する。
/Zp
構造体メンバの配置、パッキング(/Zp{1|2|4|8|16})
/Zs
構文をチェックするだけ。
